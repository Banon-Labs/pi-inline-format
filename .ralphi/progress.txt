## Codebase Patterns
- Keep the Pi-facing TypeScript wrapper minimal and move transcript analysis contracts into the Rust crate.
- Document stable Rust CLI JSON contracts in `README.md` when later iterations need to consume them from TypeScript.
- Ignore `.pi/` local Pi state in git so Ralph commits stay focused on product code.
- When separating transcript wrappers from embedded code, keep wrapper text as one or more ordered outer regions instead of leaving the embedded body inside a single bash span.
- When a TypeScript wrapper can reuse `rust/target` binaries, guard that fast path with a freshness check so stale debug binaries do not silently drift from the Rust sources.
- When Pi-facing output needs separate code fences, keep the markdown formatting as a pure projection of Rust `render_blocks` so TypeScript does not re-split transcripts.
- Keep nested-language detection extensible by expressing each supported transcript shape as a Rust pattern descriptor and leaving the `regions`/`render_blocks` contract unchanged.

- For normal Pi user-flow proof, override the built-in `bash` tool renderer and feed it Rust-derived render metadata from the command text so helper commands are not the only validation path.
